## 職涯
### 今年的成長
- 心態：
  - 就像早餐店阿姨都會喊的帥哥，獵頭總會把你在現職的長年經歷描述成稀缺的資產，但那十年到底是頭一年用了十年，還是每一年都有新的理解。

    這也讓我開始問自己，現在做的是「技術負責人」還是更像「技術領袖」，以及如何在這兩者之間找到自己的位置。

    搞技術如果沒前途，那就把焦點放在人際與判斷力；朋友在面試時提到主管在資料庫架構上的保守態度，也提醒我對於新領域的理解與開放性，就算沒有了員工識別證，也要持續做人才經營。

- 能力：
  - 重溫自己的技術棧後我才發現，能證明的不是自己在公司的名片，而是能直接做出東西。

    因此我在 Vibe Coding 的實作裡把產出截成可被看見的作品，也開始整理零碎的想法，把 數位名片 和 Blog 系統化，滿足我只想寫幾篇簡單文章的習慣。

    沒有了公司的識別證，你還剩下什麼？對我來說，是這些公開的產出和持續練習的過程。

- 人際：
  - 透過 [Linkedin](https://www.linkedin.com/in/wnlin/) 和中國的 `DB 信息差` 社群，累積到一些不同的視角和案例，在公司外部的人際經營會更省力些。

    既然可以踩在巨人的肩膀，就不再封閉自己去摸索新的可能。

### 今年的作品／成果 與 思考
- 工作專案：
  - 為了讓不同產品線的特性能落地新的資料庫架構，完成了一些把資料庫從單體叢集搬到分散式架構的 PoC 再一次驗證 ACID 系統下 Multiple Primary 就是砸自己的腳而已；
  
    也在探索中問自己：這解決了哪些痛點？哪些產品可以承受這種轉型？即便現階段大家都有 (Active/Standby) DR 的架構與概念了，真正的 Active/Active 才是下一步？

    目前的分散式部署狀態還像理論版的準備階段，技術投資的 ROI 要與企業的成熟度對齊。我認為答案不是只有更先進的技術，後勤資源（像是 TiDB 在新加坡與中國的支持）才是台灣的企業導入前的關鍵風險指標。

- 個人項目：
  - Vibe Coding 的精神也體現在這些實驗中，研究相關技術棧時反而更投入，兩次十美元的「儲值」都毫不猶豫。
  - 也滿足了現在很少維護「同一個 App 給大家用」，而是讓每個人有屬於自己習慣的 interface 的趨勢。
  - 免費的技術棧讓我每次 commit 都可以即刻更新靜態站台與 Blog，
    - [數位名片](https://wnlin.org)
    - [Blog](https://blog.wnlin.org)
  - [體系化的知識庫](https://github.com/heavenruler/dba_career/tree/master/KnowledgeBase)
    - 彙整及產出，目前用的上手的還是 [NotebookLM](https://notebooklm.google.com/) 與 [ima.copilot](https://ima.qq.com/) 用得上手些，接下來就會是知識體系的資訊與內容產出。

- 失敗清單：
  - 拖延症讓我把腳步放慢，這才清楚看見在 K8s 上營運資料庫需要的架構規劃、產品端對 self-management 的要求，才發現卡住我的不是技術細節，而是落實這些配套的方式。又再一次驗證了，難搞的永遠不是技術。
   
- 留下個問題：
  - 數年前還會主動做地區性風險的預防規劃，今年反而就沒那麼多風采了。
